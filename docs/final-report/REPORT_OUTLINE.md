## 功能性需求

| 序号 | 需求 | 当前实现 |
| --- | --- | --- |
| F01 | 系统参数维护 | `/params`，支持查询、新增、修改、删除普通参数，保护 `UNITACCNUM/PERACCNUM`。 |
| F02 | 单位开户 | `/units/open`，生成 12 位单位账号，写入 `TB002`，递增 `TB001.UNITACCNUM`。 |
| F03 | 个人开户 | `/persons/open`，支持手工开户、销户账户重启、单位汇总更新。 |
| F04 | 个人 Excel 批量开户 | `/persons/open` 页面内上传 Excel，全部成功才提交，失败显示逐行原因。 |
| F05 | 单位资料修改 | `/units/edit`，只修改指导书允许字段，销户单位不可修改。 |
| F06 | 个人资料修改 | `/persons/edit`，支持普通修改、身份证冲突提示、强制变更新建错误账户。 |
| F07 | 单位信息查询 | `/units/query`，支持单位账号精确查询和单位名称模糊查询。 |
| F08 | 个人信息查询 | `/persons/query`，支持个人账号和证件号码查询。 |

## 非功能性需求

| 类别 | 要求 | 项目实现 |
| --- | --- | --- |
| 架构 | MVC、Spring、MyBatis 分层 | Controller 负责请求，Service 负责业务和事务，Mapper 负责 SQL，JSP 负责页面。 |
| 数据一致性 | 开户、序号生成、汇总更新必须事务安全 | Service 层使用事务；账号序号 `SELECT ... FOR UPDATE` 后递增。 |
| 数据类型 | 金额字段不能使用浮点数 | Java 使用 `BigDecimal`，数据库使用 `DECIMAL(16,2)`。 |
| 安全配置 | 仓库不提交真实数据库密码 | `jdbc.properties` 保持 `change_me`，本地用被忽略的 `jdbc-local.properties`。 |
| 可验收性 | 可通过 Maven、Tomcat、MySQL、浏览器页面验证 | 已完成 `mvn clean package`、真实 MySQL 初始化、Tomcat WAR 部署和入口验证。 |

## 角色说明

| 角色 | 说明 |
| --- | --- |
| 综合柜员 | 办理单位开户、个人开户、资料修改、信息查询和系统参数维护的主要操作者。 |
| 缴存单位经办人 | 提交单位开户资料、核对单位开户或资料变更回单。 |
| 缴存职工 | 个人账户信息对应主体，通过柜员办理个人开户和资料变更。 |
| 系统管理员/运维人员 | 初始化数据库、配置本地数据库连接、部署 Tomcat WAR。 |

## 业务流程说明

1. 单位开户：录入单位资料 -> 校验组织机构代码和比例 -> 锁定 `TB001.UNITACCNUM` -> 生成单位账号 -> 写入 `TB002` -> 序号加 1 -> 返回开户回单。
2. 个人开户：输入正常单位账号 -> 反显单位资料 -> 校验个人资料 -> 锁定 `TB001.PERACCNUM` -> 生成或重启个人账号 -> 写入 `TB003` -> 更新 `TB002` 汇总 -> 返回个人开户回单。
3. Excel 批量开户：上传 Excel -> 解析非空行 -> 对每行复用个人开户校验 -> 如任一行失败则整体回滚 -> 全部合法时逐条开户。
4. 个人资料强制变更：目标账号提交新证件号 -> 发现被其他账号占用 -> 回显占用账户 -> 确认强制 -> 新建错误账户保留占用账户原信息 -> 释放占用账户证件号 -> 更新目标账户。

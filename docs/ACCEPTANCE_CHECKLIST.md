# 指导书对齐验收清单

## 环境与数据库

- 使用 JDK 25、Maven 3.9.16、Tomcat 9、MySQL。
- `src/main/resources/jdbc.properties` 部署前已按本机 MySQL 修改，且不提交真实密码。
- 需要重新导入课程设计测试库时执行：

```bat
mysql -u root -p < db/schema.sql
mysql -u root -p housingfund_collection < db/data.sql
mysql -u root -p housingfund_collection < db/demo-data.sql
```

- 执行构建：

```bat
D:\dev\apache-maven-3.9.16\bin\mvn.cmd clean package
```

期望 `BUILD SUCCESS`，测试失败数和错误数均为 0。

## 2.2.1 系统参数维护

- 访问 `/params`。
- 查询 `UNITACCNUM` 和 `PERACCNUM`。
- 新增普通系统参数，字段包含键值信息、当前序号、最大序号、描述、备注1。
- 修改普通系统参数，确认不能篡改主键 `SEQNAME`。
- 删除普通系统参数。
- 尝试删除 `UNITACCNUM`，应提示禁止删除。
- 尝试删除 `PERACCNUM`，应提示禁止删除。
- 新增或修改时 `SEQ > MAXSEQ` 应失败。

## 2.2.2 单位开户

- 访问 `/units/open`。
- 页面只出现单位名称、单位地址、组织机构代码、单位类别、企业类型、发薪日期、联系电话、单位经办人、经办人身份证号码、单位比例、个人比例、备注。
- 使用合法数据开户，成功后显示《公积金开户回单》。
- 验证生成 12 位单位公积金账号。
- 验证 `TB001.UNITACCNUM.SEQ` 成功后加 1。
- 重复正常状态组织机构代码时提示“该单位已建户”。
- 单位名称超过 50 个汉字应失败。
- 组织机构代码不是 9 位应失败。
- 发薪日期不在 `01-31` 应失败。
- 经办人身份证号码非法应失败。
- 单位比例小于 `0.050` 应失败。
- 个人比例大于 `0.120` 应失败。
- 序号超过最大值应失败。

## 2.2.3 个人开户

- 访问 `/persons/open`。
- 输入正常状态单位公积金账号后反显单位名称、单位比例、个人比例。
- 页面只出现单位公积金账号、单位名称、单位比例、个人比例、姓名、证件类型、证件号码、缴存基数。
- 使用合法数据开户，成功后显示《个人住房公积金开户回单》。
- 验证生成 12 位个人公积金账号。
- 验证 `TB001.PERACCNUM.SEQ` 新开户成功后加 1。
- 单位不存在应失败。
- 单位已销户应失败。
- 证件号码已存在且非销户应提示“该人员已开户”。
- 证件号码已存在且销户时应重新启用原个人账户，不生成新账号，不更新 `PERACCNUM.SEQ`。
- 姓名超过 12 个汉字应失败。
- 证件类型不是 `01身份证` 应失败。
- 缴存基数小于等于 0 应失败。
- 单位比例、个人比例从单位资料取值，后端仍校验 `0.050-0.120`。
- 验证单位缴存基数、单位月应缴额、个人月应缴额、单位人数正确增加。

## 2.2.4 单位资料修改

- 访问 `/units/edit`。
- 根据单位公积金账号反显单位资料。
- 页面只允许修改单位名称、单位地址、组织机构代码、单位类别、企业类型、发薪日期、联系电话、单位经办人、经办人身份证号码、备注。
- 不修改任何字段应失败。
- 修改前后完全一致应失败。
- 已销户单位不能修改。
- 修改后的组织机构代码与单位名称同时重复时应失败。
- 后端不可通过请求参数篡改单位公积金账号、单位比例、个人比例、余额、缴存基数、人数、月缴额、状态、最后汇缴月、机构代码、柜员、建立日期。
- 成功后显示《住房公积金账户资料变更回单》。

## 2.2.5 个人资料修改

- 访问 `/persons/edit`。
- 根据个人公积金账号反显个人资料、单位公积金账号、单位名称。
- 页面只允许修改姓名、证件类型、证件号码。
- 不显示或提交个人联系方式等非指导书字段。
- 不修改任何字段应失败。
- 已销户个人不能修改。
- 后端不可通过请求参数修改个人公积金账号、单位比例、个人比例、余额、月缴额、状态、最后汇缴月等不可修改字段。
- 修改证件号码与系统中其他账户冲突时，应回显占用个人公积金账号、证件号码、姓名、状态、单位名称、单位公积金账号。
- 冲突时应提示是否强制变更。
- 强制变更时，占用账户证件号码首位改为 `9`，其余位数不变；当前账户更新为正确姓名、证件类型、证件号码。
- 当前实现未额外新建一个人账户保存错误信息，原因是课程设计固定 `TB003.IDCARD` 唯一且缺少错误账户迁移目标表；当前处理方式已完成冲突回显、强制变更提示、占用账户证件号码首位改 `9`、当前账户更新。

## 2.2.6 单位信息查询

- 访问 `/units/query`。
- 输入单位公积金账号可精确查询单位情况。
- 输入单位名称可模糊查询。
- 模糊查询结果中，单位名称可点击或双击后显示该单位情况。
- 输出包含单位名称、单位公积金账号、单位地址、经办人姓名、联系电话、公积金余额、单位比例、个人比例、合计比例、最后汇缴月、单位月应缴额、个人月应缴额、合计月缴额、人数。

## 2.2.7 个人信息查询

- 访问 `/persons/query`。
- 输入个人公积金账号可查询。
- 输入证件号码可查询。
- 输出包含缴存单位全称、缴存单位公积金账号、姓名、个人公积金账号、余额、开户日期、最后汇缴月、单位比例、个人比例、合计比例、单位月应缴额、个人月应缴额、合计月缴额。

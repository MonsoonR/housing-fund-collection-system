# 手动验收清单

## 1. 环境检查

- 执行 `java -version`，确认 JDK 可用。
- 执行 `mvn -v`，确认 Maven 可用。
- 确认 MySQL 已启动。
- 确认 Tomcat 9 已启动。

## 2. 数据库初始化

1. 执行 `db/schema.sql`。
2. 执行 `db/data.sql`。
3. 可选执行 `db/demo-data.sql`，导入答辩演示数据。

## 3. WAR 构建

命令：

```bat
D:\dev\apache-maven-3.9.16\bin\mvn.cmd clean package
```

期望：

```text
BUILD SUCCESS
```

## 4. Tomcat 部署

1. 将 `target/housingfund-collection.war` 放入 Tomcat `webapps`。
2. 启动 Tomcat。
3. 访问 `http://localhost:8080/housingfund-collection/`。

## 5. 七个核心模块逐项验收

### 系统参数维护

- 访问 `/params`。
- 查询 `UNITACCNUM`。
- 新增普通参数。
- 修改普通参数。
- 删除普通参数。
- 尝试删除 `UNITACCNUM`，应被拒绝。

### 单位开户

- 访问 `/units/open`。
- 使用合法数据开户。
- 验证生成 12 位单位账号。
- 重复组织机构代码应提示“该单位已建户”。
- 比例小于 5% 或大于 12% 应失败。

### 个人开户

- 访问 `/persons/open`。
- 输入单位账号后反显单位名称、单位比例、个人比例。
- 使用合法数据开户。
- 验证生成 12 位个人账号。
- 重复身份证号非销户应提示“该人员已开户”。
- 销户个人应能重新启用。

### 单位资料修改

- 访问 `/units/edit`。
- 输入单位账号反显资料。
- 修改单位名称或联系电话。
- 不修改任何字段应失败。
- 已销户单位不能修改。

### 个人资料修改

- 访问 `/persons/edit`。
- 输入个人账号反显资料。
- 修改姓名或证件号码。
- 不修改任何字段应失败。
- 身份证冲突时应回显占用账户信息。
- 强制变更时，占用账户身份证号首位改为 `9`。

### 单位信息查询

- 访问 `/units/query`。
- 按单位账号精确查询。
- 按单位名称模糊查询。
- 检查单位名称、账号、地址、经办人、电话、余额、缴存比例、最后汇缴月、月汇缴金额、人数是否显示。

### 个人信息查询

- 访问 `/persons/query`。
- 按个人账号查询。
- 按身份证号查询。
- 检查单位名称、单位账号、姓名、个人账号、余额、开户日期、最后汇缴月、缴存比例、月汇缴金额是否显示。

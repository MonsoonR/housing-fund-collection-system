# 答辩演示顺序

1. 首页展示 7 个核心模块。
2. 系统参数维护：展示 `UNITACCNUM`、`PERACCNUM`。
3. 单位开户：新建单位并展示回执。
4. 个人开户：基于刚才单位新建个人并展示回执。
5. 单位信息查询：查刚才单位。
6. 个人信息查询：查刚才个人。
7. 单位资料修改：修改单位联系电话或单位名称。
8. 个人资料修改：修改个人姓名或证件号码。
9. 展示错误校验：
   - 重复组织机构代码。
   - 重复身份证号。
   - 比例越界。
   - 不修改任何字段。
10. 最后说明项目采用 SSM 分层：
    - Controller 接收请求。
    - Service 处理业务和事务。
    - Mapper 操作数据库。
    - JSP + JavaScript 完成页面和前端校验。
